1. 프로젝트 개요
자바의 객체지향 프로그램(OOP) 핵심 원리를 이해하기 위해 제작한 간단한 카페 주문 관리 시스템입니다.

2. 핵심 학습 내용
추상화 (Abstraction)
- Menu 클래스를 abstract로 선언하여 불완전한 설계도를 만들고, 자식 클래스들이 반드시 구현해야 할 가이드라인(prepare())을 제시했습니다.
- 공통 속성인 name과 price를 부모 클래스에 정의하여 코드 중복을 제거했습니다.

상속과 오버라이딩 (Inheritance & Overriding)
- Coffee, Dessert 클래스가 Menu를 상속받아 각 메뉴 특성에 맞는 조리 과정을 구현했습니다.
- @Override 어노테이션을 사용하여 부모의 메서드를 정확히 재정의했는지 컴파일 시점에 체크하도록 설정했습니다.

접근 제어자 (protected)
- 부모 클래스의 필드를 protected로 설정하여, 외부의 접근은 차단하면서도 상속 관계에 있는 자식 클래스에서는 자유롭게 부모의 변수를 사용할 수 있도록 설계했습니다.

다형성 (Polymorphism)
- ArrayList<Menu>를 활용하여 Coffee와 Dessert 객체를 하나의 리스트(부모 타입 바구니)에 담아 관리했습니다.
- Upcasting : 자식 객체가 부모 타입으로 자동 형변환되어 리스트에 저장되는 원리를 이해했습니다.
- 반복문을 통해 하나의 부모 리모컨(Menu m)으로 서로 다른 객체의 메서드를 일괄 호출하는 다형성의 효율성을 체감했습니다.

3. 실행 흐름
- 1. ArrayList<Menu> 생성
- 2. 다양한 매뉴 객체(Coffee, Dessert)를 생성하여 리스트에 add
- 3. 향상된 for 문을 사용하여 전체 메뉴 정보 출력 및 prepare() 실행
- 4. 모든 메뉴의 가격을 합산하여 최종 결제 금액 출력
